# JavaScript DOM Interview Preparation

DOM interviews test whether you can **manipulate pages without React** — select elements, handle events, build small interactive UIs, and understand how the browser works.

## What Interviewers Look For

- Selecting and traversing the DOM
- Event handling (bubbling, delegation, preventDefault)
- Creating/updating elements dynamically
- Form handling and validation
- Basic performance awareness (reflows, delegation)
- Fetch + DOM integration (mini projects)

1. `topics-not-to-miss.md` — identify gaps
2. `interview-questions.md` — 5 questions/day
3. `mini-dom-challenges.md` — build all 8 in plain HTML/CSS/JS
4. `project-style-questions.md` — practice explaining tradeoffs
5. `mock-interview.md` — practice with a friend
6. `revision-sheet.md` — day before interview

## Time Estimate

- **Minimum viable prep:** 3–4 days
- **Solid prep:** 1 week (with all mini challenges)

## Project Connection

Be ready to demo: a todo app, form validation, dynamic list, or API-driven UI you built without a framework. Explain event delegation and how you structured your JS files.
