# React Interview Preparation

React interviews test whether you can **build and reason about component-based UIs** — state, hooks, rendering, and real project decisions.

## What Interviewers Look For

- Component design (smart vs presentational)
- Hooks used correctly (not just copied)
- State management patterns
- Performance awareness (memo, callback, list keys)
- How React fits your actual project

## Study Order

1. `topics-not-to-miss.md` — identify gaps
2. `hooks-deep-dive.md` — understand each hook deeply
3. `interview-questions.md` — 5 questions/day, out loud
4. `performance.md` — know when to optimize (and when not to)
5. `project-style-questions.md` — practice scenarios
6. `mock-interview.md` — practice with a friend
7. `revision-sheet.md` — day before interview

## Time Estimate

- **Minimum viable prep:** 1 week (if you know JS well)
- **Solid prep:** 2 weeks (with a small React project)

## Project Connection

Be ready to explain: component structure, where state lives, how you fetch data, routing choice, and one performance or bug fix you made.
