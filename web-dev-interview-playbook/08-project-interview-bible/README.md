# Project Interview Bible

Your project is your **strongest interview asset** as a student. Technical rounds often spend 15–30 minutes on it. This section helps you explain it clearly at any depth.

## Why This Matters

Interviewers ask about your project to check:
- Did you actually build it (or just clone a tutorial)?
- Do you understand the architecture you chose?
- Can you talk about tradeoffs and what you'd improve?
- Will you be able to contribute on day one?

A mediocre project explained well beats an impressive project explained poorly.

## The Project Explanation Formula

Every good project explanation includes:

1. **Problem** — What problem does it solve? For whom?
2. **Solution** — What did you build? Demo in one sentence.
3. **Tech stack** — Why these choices?
4. **Your role** — What did YOU specifically do?
5. **Challenge** — One hard problem you solved.
6. **Result** — Users, metrics, or what you learned.
7. **Next step** — What you'd improve (shows growth mindset).

## Before Every Interview

- [ ] 2-minute pitch rehearsed out loud
- [ ] Can draw architecture on whiteboard in 2 minutes
- [ ] Know your hardest bug and how you fixed it
- [ ] Have project deployed and ready to screen-share
- [ ] GitHub repo is clean (README, no secrets in code)
