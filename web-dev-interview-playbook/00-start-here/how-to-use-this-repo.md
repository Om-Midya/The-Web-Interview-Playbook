# How to Use This Repository

This is a **playbook**, not a novel. You don't read it cover to cover — you use it like a training manual.

## Step 1: Know Your Interview Type

| Interview Type | Start Here |
|----------------|------------|
| Campus placement (service company) | HTML/CSS → JS Core → Project Bible → HR |
| Startup internship | JS Core → React → Project Bible → System Design basics |
| Product company (frontend) | React → JS Core → DOM → Performance |
| Product company (full-stack) | JS → React → Node → Project Bible → System Design |
| HR / Managerial round | `09-hr-interviews/` only (but prepare project story first) |

## Step 2: Use the Question Format Actively

Every question in this repo follows a structure. Don't just read the **Answer** — practice:

1. **Close the answer** and explain it in your own words (out loud)
2. **Anticipate follow-ups** before reading them
3. **Connect to your project** — replace generic examples with your actual work
4. **Note common mistakes** — these are what interviewers listen for

## Step 3: The Project Bible Is Non-Negotiable

In 70% of student interviews, the interviewer spends significant time on **your project**. If you can't explain:

- Why you built it
- Why you chose your stack
- One hard problem you solved
- What you'd improve

...then 50 JavaScript questions won't save you.

Spend **equal time** on `08-project-interview-bible/` and technical sections.

## Step 4: Mock Interviews Beat Passive Reading

| Activity | Value |
|----------|-------|
| Reading answers silently | Low |
| Writing answers in a notebook | Medium |
| Explaining answers out loud | High |
| 45-min mock with a friend | Very High |
| Recording yourself answering | High (and slightly painful — do it anyway) |

Use `10-mock-interview-kits/` — each kit is a timed script.

## Step 5: Revision Sheets Before the Interview

One day before your interview, only read:

- `revision-sheet.md` files for your weak topics
- Your project 2-minute pitch
- `00-start-here/answer-frameworks.md`
- Top 10 questions from `11-interview-patterns/top-why-questions.md`

## What "Prepared" Looks Like

You're ready when you can:

- [ ] Answer "Tell me about yourself" in 90 seconds without umm-ing
- [ ] Draw your project architecture on paper in 3 minutes
- [ ] Explain 5 concepts from your resume if asked "why did you use X?"
- [ ] Handle "I don't know" gracefully (see `answer-frameworks.md`)
- [ ] Complete a mock interview without checking notes

Start with the [ROADMAP](../ROADMAP.md). Adjust based on your interview date.
