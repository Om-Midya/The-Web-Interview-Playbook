# Web Dev Interview Playbook

A structured, student-friendly interview preparation guide for **2nd and 3rd-year engineering students** targeting web development roles — internships, campus placements, and junior full-stack positions.

## Why This Repo Exists

Most interview prep resources are either:

- Random question dumps with no context
- LeetCode-heavy (great for DSA, weak for web dev)
- Tutorial blogs that don't explain *why* interviewers ask something

This playbook is different. Every section teaches you **how to think in an interview**, not just what to memorize.

## How to Use This Repo

1. **Start with** [how-to-use-this-repo.md](./00-start-here/how-to-use-this-repo.md)
2. **Follow the** [4-week ROADMAP](./ROADMAP.md)
3. **Pick your weak areas** — don't read everything linearly
4. **Practice out loud** — reading silently is not interview practice
5. **Use mock interview kits** with a friend or record yourself

## Who Is This For?

- students in 2nd or 3rd year
- Self-taught developers preparing for first internships
- Anyone switching to web dev from another branch
- Students who have a project but struggle to explain it in interviews

## What This Repo Is NOT

- Not a DSA sheet (use LeetCode/Codeforces for that)
- Not a full tutorial course (use freeCodeCamp, MDN for deep learning)
- Not company-specific
