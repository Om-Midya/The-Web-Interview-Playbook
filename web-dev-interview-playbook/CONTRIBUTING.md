# Contributing to Web Dev Interview Playbook

Thank you for helping the next batch of students. This repo grows better when real interview experiences and improved answers come from the community.

## Ways to Contribute

### 1. Add an Interview Experience (Most Valuable)

1. Copy `12-interview-experiences/experience-template.md`
2. Rename it: `company-name-role-year.md` (e.g., `razorpay-intern-2025.md`)
3. Fill in every section honestly — include questions you couldn't answer
4. Open a Pull Request

**What makes a great experience post:**
- Actual questions asked (not "they asked JS questions")
- What follow-ups came after your answers
- How much time was spent on your project
- What you'd do differently

### 2. Improve Existing Answers

- Fix factual errors
- Add clearer examples
- Simplify explanations for beginners
- Add follow-up questions you were actually asked

### 3. Add Questions

If you were asked something not in the repo:
- Add it to the right section's `interview-questions.md`
- Use the standard question format (see below)
- Include difficulty, why interviewers ask, follow-ups, mistakes, project connection

### 4. Report Issues

- Broken links
- Outdated info (e.g., Next.js API changes)
- Typos

## Pull Request Guidelines

1. **One topic per PR** — don't mix HR questions with React fixes
2. **Keep tone student-friendly** — no corporate jargon
3. **No copy-paste from paid courses** — original or properly attributed
4. **Test code snippets** — if you add code, make sure it runs
5. **No company confidential info** — don't share internal questions marked NDA

## Code of Conduct

- Be respectful in PR reviews
- Don't gatekeep ("this is too easy")
- Remember: someone else's "basic" question might be another student's gap

## File Naming

- Interview experiences: `company-role-year.md` (lowercase, hyphens)
- No spaces in filenames
- Markdown only (`.md`)

## Questions?

Open an issue with the `question` label. We'll respond when we can.
