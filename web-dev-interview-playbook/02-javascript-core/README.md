# JavaScript Core Interview Preparation

JavaScript interviews test whether you understand **how the language actually runs** — not just syntax. Expect output questions, polyfill challenges, and "explain this code" moments.

## What Interviewers Look For

- Closures, scope, and the `this` keyword
- Event loop, microtasks vs macrotasks
- Prototypes and inheritance
- ES6+ features used correctly (not just copied from Stack Overflow)
- Ability to implement common utilities from scratch

## Study Order

1. `topics-not-to-miss.md` — identify gaps
2. `interview-questions.md` — 5 questions/day, out loud
3. `tricky-output-questions.md` — predict output before scrolling to answers
4. `polyfills.md` — write each polyfill without looking, then compare
5. `coding-questions.md` — solve 2–3 problems per day
6. `mock-interview.md` — practice with a friend
7. `revision-sheet.md` — day before interview

## Time Estimate

- **Minimum viable prep:** 5–7 days
- **Solid prep:** 2 weeks (with polyfills + coding problems)

## Project Connection

Be ready to explain: async data fetching in your project, how you handled state, any utility functions you wrote, and one bug you debugged that taught you something about JS internals.
